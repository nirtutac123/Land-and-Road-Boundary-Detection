
COMPARATIVE ANALYSIS RESULTS

1. PolyFit Baseline
   - Approach: cubic polynomial fitting to lane points.
   - Strength: fast and easy to explain.
   - Limitation: depends on clean lane-point geometry and struggles when lanes are missing or occluded.

2. CNN-Style Segmentation
   - Approach: pixel-level lane-mask prediction in a reduced executable demo.
   - Strength: can represent dense lane evidence rather than only curve parameters.
   - Limitation: a full production CNN would require proper training, validation, and tuning.

3. Classical Edge Detection
   - Approach: Canny edges plus Hough line transform.
   - Strength: no training required and useful as a transparent baseline.
   - Limitation: sensitive to lighting, shadows, glare, and line-marking quality.

MAIN RESULT FROM THIS RUN

- Best F1-score: PolyFit (0.6093)
- Best mean IoU: PolyFit (0.4394)
- Best precision: PolyFit (0.4518)

ERROR ANALYSIS AND LIMITATIONS

- Occlusion can hide lane boundaries and reduce recall.
- Night, glare, and shadow scenes can create false or missing edges.
- No-line and crossroad cases are difficult because the target lane geometry is ambiguous.
- The notebook uses a reduced demonstration path; a final production model should use a stronger trained segmentation architecture and a standard lane-detection benchmark metric.

RECOMMENDATIONS

- Keep PolyFit as the interpretable baseline.
- Use the classical detector as a fast traditional computer-vision comparison.
- Treat CNN segmentation as the most promising direction for a stronger final system, but train and validate it properly if more time/compute is available.
- Present both the metric table and visual examples, because lane detection quality is spatial and easy to misread from a single score.

PROJECT CONCLUSION

The project question is answered by comparing simple geometry, classical computer vision, and segmentation-style detection on the same CULane validation workflow. The most defensible conclusion is that no single lightweight method handles every scenario perfectly: transparent baselines are useful for speed and explanation, while segmentation is the better direction for difficult real-world scenes.
